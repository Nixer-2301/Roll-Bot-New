# Roll-Bot — 积分助手 v2.2.0

基于 [khl.py](https://github.com/TWT233/khl.py) 开发的 KOOK（开黑啦）机器人。

## 功能

### 基础命令

| 命令 | 说明 |
|---|---|
| `/roll [最小值] [最大值] [数量]` | 随机数抽取，支持自定义范围和数量，卡片返回 |
| `/cpu` | 查看服务端 CPU 使用率及物理/逻辑核心数 |
| `/help [页数]` | 卡片分页展示命令帮助，按钮翻页 |
| `/info` | 版本、作者及 khl.py 链接 |
| `/music 歌名 歌手` | 倾听音乐状态（网易云） |

### 签到 & 积分

| 命令 | 说明 |
|---|---|
| `/签到` | 每日签到，获得 1-10 随机积分，连续 7 天额外奖励 5-20 分 |
| `/积分` | 查看当前积分、连续签到、累计签到 |
| `/排行榜 [人数]` | 积分排行 Top N（默认 10，最多 20） |
| `/转账 @用户 数量` | 向其他用户转积分 |

### 管理员命令

| 命令 | 说明 |
|---|---|
| `/补签 @用户` | 为用户补签当天 |
| `/签到查询 @用户` | 查看用户签到/积分数据 |
| `/重置签到 @用户` | 清空用户所有签到记录 |
| `/签到设置 @用户 连续 累计 [日期]` | 修改用户签到字段 |
| `/设置积分 @用户 数量` | 设定用户积分值 |

### 权限判断

管理员身份通过以下方式判定（任一满足即可）：
- 用户 ID 在 `ADMIN_IDS` 白名单中
- 用户是服务器主（guild master）
- 用户拥有「管理员」角色或 `Administrator` 权限位

## 运行要求

- Python 3.8+
- [KOOK 开发者平台](https://developer.kookapp.cn/) 注册的机器人 Token

## 安装

```bash
pip install khl.py psutil aiosqlite
```

## 配置

1. 将 `bot = Bot(token='...')` 中的 token 替换为你的机器人 Token
2. 将 `ADMIN_IDS` 集合中的 ID 替换为你的 KOOK 用户 ID

## 数据存储

使用 **SQLite**（`checkin.db`），自动创建于脚本同级目录。
